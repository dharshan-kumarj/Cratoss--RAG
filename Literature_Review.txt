<b>2. Literature Review</b>

<b>2.1 Review of Existing Methods and Related Work</b>

Retrieval-Augmented Generation (RAG) was first introduced by Lewis et al. (2020) as a general-purpose fine-tuning approach for knowledge-intensive NLP tasks. The original architecture combined a dense passage retriever (DPR) with an encoder-decoder language model, demonstrating significant improvements over purely parametric LLM approaches on benchmarks like Natural Questions and TriviaQA. Since then, RAG has evolved through several generations: from the basic dense retrieval + generation pipeline to more sophisticated approaches incorporating hybrid retrieval, reranking mechanisms, and confidence calibration. Research by Gao et al. (2023) categorized RAG systems into four types—Pre-RAG, Mid-RAG, Post-RAG, and Autonomous-RAG—based on where retrieval occurs in the generation process, highlighting the importance of retrieval timing and quality in determining overall system performance.

The field of information retrieval has long relied on term-based methods such as BM25 (Best Matching 25), introduced by Robertson and Zaragoza (2009), which remains a strong baseline for keyword-based search due to its theoretical foundation in probabilistic relevance theory. BM25's term frequency saturation and document length normalization make it particularly effective for exact keyword matching in technical domains where precise terminology is critical. However, BM25 alone cannot capture semantic relationships between terms, leading to the development of dense retrieval methods using neural embeddings. The Sentence-BERT (SBERT) model by Reimers and Gurevych (2019) revolutionized semantic search by producing semantically meaningful sentence embeddings that enable cosine similarity-based retrieval, forming the backbone of modern RAG systems.

Cross-encoder reranking, popularized by Nogueira et al. (2019) in the context of document re-ranking, represents a critical advancement in retrieval pipelines. Unlike bi-encoder approaches that encode queries and documents independently, cross-encoders process query-document pairs jointly, allowing for deeper interaction between query terms and document content. The MS MARCO benchmarks, released by Microsoft Research, provided standardized evaluation datasets that accelerated research in neural ranking models, leading to models like ms-marco-MiniLM-L-6-v2 that balance effectiveness with computational efficiency. Hybrid retrieval combining sparse methods like BM25 with dense semantic search, often fused using Reciprocal Rank Fusion (RRF) as proposed by Webber et al. (2010), has emerged as a best practice for maximizing retrieval coverage across different query types.

In the domain-specific RAG space, several notable projects have explored specialized implementations. LangChain, a popular framework for building LLM applications, provides modular components for RAG implementation but often lacks the sophisticated retrieval strategies needed for production systems. ChromaDB and other vector databases (Pinecone, Weaviate, Milvus) have emerged as essential infrastructure for storing and searching high-dimensional embeddings at scale. The combination of hybrid retrieval with cross-encoder reranking has been shown in multiple studies to outperform either approach alone, with research by Tang et al. (2023) demonstrating that two-stage retrieval pipelines can improve precision by 15-30% over single-stage dense retrieval on technical document QA tasks.

<b>2.2 Comparison with Similar Projects and Existing Solutions</b>

When comparing RAG-Cratoss with existing solutions, several key dimensions emerge as critical for evaluation: retrieval methodology, confidence handling, domain specificity, and deployment architecture. Commercial RAG platforms like Azure AI Search, Amazon Kendra, and Google Vertex AI Search offer managed RAG capabilities but typically operate as black-box cloud services with limited transparency in retrieval mechanisms and significant per-query costs. These platforms, while user-friendly, do not provide the granular control over hybrid retrieval parameters, reranking thresholds, or confidence tier definitions that domain-specific applications require. Furthermore, their cloud-based nature introduces latency, privacy concerns, and dependency on external service availability.

Open-source alternatives like LangChain + ChromaDB implementations provide more flexibility but often implement only basic semantic retrieval without hybrid search or sophisticated reranking. The LlamaIndex framework offers more advanced retrieval strategies including vector search, keyword search, and recursive retrieval, but still lacks the integrated confidence-tiered response generation that prevents hallucinations in high-stakes domains. Most existing open-source RAG tutorials demonstrate toy examples with generic document collections, failing to address the specific challenges of technical documentation where exact terminology matching is as important as semantic understanding.

RAG-Cratoss distinguishes itself through several design choices that address these gaps. First, the implementation of BM25 + semantic hybrid retrieval with RRF fusion ensures that both exact technical term matching and semantic similarity contribute to retrieval quality. Second, the cross-encoder reranking layer provides a second stage of precision refinement that dramatically improves result quality over the initial retrieval set. Third, and most importantly, the 3-tier confidence evaluation system represents a novel approach to response calibration that actively prevents hallucination rather than merely hoping for accurate generation. This confidence-tiered architecture allows the system to distinguish between high-confidence document-grounded answers, uncertain answers with weak context, and out-of-domain queries that should be refused or handled through controlled fallback.

The local-only deployment architecture also differentiates RAG-Cratoss from most alternatives. By running Llama 3.2 entirely via Ollama, the system achieves complete data privacy, eliminates API costs, and enables deployment in air-gapped environments. While this approach requires upfront hardware investment, it provides long-term cost advantages for organizations processing large volumes of queries. The modular design allows individual components (retriever, reranker, confidence checker, generator) to be upgraded or replaced independently, future-proofing the system against rapid advances in embedding models or LLM architectures.

In summary, while existing RAG frameworks and commercial platforms provide valuable building blocks, RAG-Cratoss synthesizes these approaches into a coherent, domain-optimized system specifically designed for technical documentation question-answering. The combination of hybrid retrieval, cross-encoder reranking, and confidence-aware generation addresses the unique requirements of IoT and network security domains where accuracy is paramount and hallucination is unacceptable.
